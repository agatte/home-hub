PHASE 1.5 — SMART AUTOMATION (Complete)
──────────────────────────────────────────

  BACKEND:
  ✅ Hue CLIP API v2 service (hue_v2_service.py)
     - Direct HTTPS calls to bridge for native scene management
     - Native scenes visible to Alexa and Hue app
     - Dynamic effects: candlelight, fireplace, sparkle, prism, glisten, opal
     - v1↔v2 light ID mapping cache
     - GET /api/scenes now returns both presets AND native bridge scenes
     - POST /api/scenes/effects/{name} activates dynamic effects
     - POST /api/scenes/effects/{name}/light/{id} per-light effects

  ✅ Automation engine (automation_engine.py)
     - Per-light control: each light can be set independently per mode
     - Time-aware activity states: brightness adapts to day/evening/night
     - Config-driven time rules (wake, ramp, evening, wind-down hours)
     - Weekday/weekend schedules with separate defaults
     - Activity overrides: gaming, working, watching, social, relax, movie
     - Party sub-modes: color_cycle, club, rave, fire_and_ice
     - Per-mode brightness multipliers (30%-150%, configurable from Settings)
     - Sleep detection: 10:30pm + 15min idle → pause YouTube + fade lights
     - Screen sync: bedroom lamp matches projector content (mss capture)
     - Manual override from dashboard (4-hour auto-timeout, optimistic UI)
     - Alexa coexistence: detects external "all off" (geofence), won't override
     - WebSocket broadcasts mode_update to all clients
     - Background loop checks every 60 seconds, auto-adapts on period transitions

  ✅ PC activity detector (pc_agent/activity_detector.py)
     - Standalone script, runs alongside server
     - psutil process detection every 15 seconds
     - Gaming: LeagueofLegends.exe, javaw.exe (OSRS), 20+ game processes
     - Working: windowsterminal, powershell, code.exe, cursor, devenv, etc.
     - Watching: VLC, Plex, media players
     - Sleep detection: after 10:30pm, 15min idle → pause media + report sleeping
     - Away: Win32 GetLastInputInfo idle >10 min
     - Reports mode changes only to /api/automation/activity

  ✅ Ambient noise monitor (pc_agent/ambient_monitor.py)
     - Blue Yeti microphone via PyAudio
     - Measures RMS volume only — never records or saves audio
     - Party detection: sustained noise >2 min + no game running = "social"
     - No false positives during gaming (headset isolates audio)
     - Auto-calibrates noise floor on startup
     - POST /api/automation/mic/calibrate endpoint

  ✅ Morning routine (morning_routine.py + scheduler.py)
     - Replaces Alexa 6:40 AM weekday routine
     - Fetches weather (OpenWeatherMap API)
     - Fetches commute traffic (Google Maps Directions API)
     - Generates TTS: "Good morning Anthony. It's 45 degrees..."
     - Plays on Sonos, triggers morning light ramp
     - AsyncScheduler: cron-like async scheduling, no external deps
     - POST /api/routines/morning/test for instant testing
     - Configurable time/volume via PUT /api/routines/morning/config
     - Volume persists in SQLite (app_settings table) across restarts

  ✅ Evening wind-down routine (winddown_routine.py)
     - Scheduled routine (default 9:00 PM)
     - Sets manual override to "relax" → triggers candlelight + dim lights
     - Lowers Sonos volume to configured level
     - Brief TTS: "Wind-down time. Lights dimmed."
     - Configurable: time, volume, candlelight on/off, weekdays-only toggle
     - POST /api/routines/winddown/test for instant testing
     - PUT /api/routines/winddown/config for settings
     - Persists config in SQLite across restarts

  ✅ Music mapper (music_mapper.py)
     - Maps activity modes to Sonos favorites (Apple Music playlists)
     - User creates playlists per mood, maps in Home Hub
     - Auto-play or suggest on mode change
     - GET /api/sonos/favorites lists available playlists
     - GET/PUT /api/sonos/music-map for mapping configuration

  ✅ Screen sync service (screen_sync.py)
     - mss screen capture → dominant color → bedroom lamp
     - EMA smoothing (alpha 0.3) + 2s transition for flicker-free
     - Captures every 2.5s, brightness clamped 15-80
     - Auto-starts in watching mode, stops on mode change

  ✅ Configurable automation schedule
     - DaySchedule + ScheduleConfig dataclasses replace hardcoded time rules
     - _build_time_rules() generates rule tuples dynamically from config
     - Separate weekday/weekend settings
     - Fields: wake_hour, wake_brightness, ramp_start_hour,
       ramp_duration_minutes, evening_start_hour, winddown_start_hour
     - Hot-reload: update_schedule_config() clears cache for immediate effect
     - Persisted in SQLite (app_settings key: "time_schedule_config")
     - Backward-compatible: filters unknown fields from stale DB data

  ✅ Per-mode brightness multipliers
     - 6 modes: gaming, working, watching, relax, movie, social
     - Range: 0.3 (30%) to 1.5 (150%), default 1.0 (100%)
     - Applied after activity state resolution, clamped to [1, 254]
     - Hot-reload: update_mode_brightness() for immediate effect
     - Persisted in SQLite (app_settings key: "mode_brightness_config")

  ✅ Settings persistence (AppSetting model)
     - Key-value JSON store in app_settings table
     - Generic load_setting() / save_setting() helpers (reusable)
     - Used by: morning routine, wind-down routine, schedule config,
       mode brightness, music discovery
     - All settings survive server restarts

PHASE 1.6 — MUSIC DISCOVERY (Complete)
──────────────────────────────────────────

  ✅ Apple Music library import (library_import_service.py)
     - Parses iTunes/Apple Music XML export
     - Aggregates artist-level stats: play count, loved count, ratings
     - Engagement filter: only imports artists with plays > 0 OR loved > 0
     - Stores in SQLite (LibraryArtist model) with genre, play stats
     - POST /api/music/library/upload — upload XML file
     - GET /api/music/library/artists — browse imported library

  ✅ Last.fm similar artist integration (recommendation_service.py)
     - Fetches similar artists from Last.fm API for seed artists
     - Caches results in DB (SimilarArtistCache) with 30-day TTL
     - Mode-specific seed selection: offsets into artist list by mode index
     - Cross-mode deduplication: excludes artists already recommended in other modes
     - Filters out artists already in user's library
     - Handles naive datetime from SQLite (normalizes to UTC)

  ✅ Mode-based recommendations
     - 6 modes: gaming, working, watching, relax, movie, social
     - Each mode gets different seed artists for variety
     - POST /api/music/recommendations/{mode}/generate — create recommendations
     - GET /api/music/recommendations/{mode} — fetch current recommendations
     - PUT /api/music/recommendations/{id}/status — accept/dismiss/save
     - Recommendations stored in DB (MusicRecommendation model) with status tracking

  ✅ Music Discovery page (MusicDiscovery.jsx)

  GET  /api/sonos/music-map                 — Mode→playlist mapping
  PUT  /api/sonos/music-map                 — Update a mapping

  Automation:

  POST /api/automation/screen-sync/stop     — Stop screen sync

  PUT  /api/automation/schedule             — Update time schedule config
  GET  /api/automation/mode-brightness      — Get per-mode brightness
  PUT  /api/automation/mode-brightness      — Update per-mode brightness

  Routines:
  GET  /api/routines                        — List scheduled routines
  POST /api/routines/morning/test           — Test morning routine

  POST /api/routines/morning/toggle         — Enable/disable morning

BUGS FIXED (2026-03-15)
================================================================================

  1. SQLite naive datetime comparison (500 error)
     - Cause: SQLite stores datetimes without timezone info; code compared
       with timezone-aware datetime.now(timezone.utc)
     - Fix: Normalize with .replace(tzinfo=timezone.utc) before comparison

  2. Identical recommendations across all modes
     - Cause: Seed artist selection always used same top 3; no cross-mode dedup
     - Fix: Offset fallback artists by mode index + exclude pending recs
       from other modes

  3. Settings volume slider resetting on page revisit
     - Cause: GET /api/routines didn't return volume; frontend hardcoded 10
     - Fix: Backend attaches service volume to task response; frontend reads it

  4. Morning routine volume not persisting across restarts
     - Cause: Volume stored only in memory on service instance
     - Fix: Persist via AppSetting model in SQLite

  5. Startup crash from stale DB config after removing away_hours
     - Cause: Saved JSON had now-removed away_start_hour/away_end_hour fields
     - Fix: Filter dict keys to valid DaySchedule fields using dataclasses.fields()

HOW TO RUN
================================================================================

  Server:
    python run.py

  PC Activity Detector (separate terminal):
    python -m backend.services.pc_agent.activity_detector

  Ambient Monitor (separate terminal, requires Blue Yeti):
    python -m backend.services.pc_agent.ambient_monitor

  Frontend dev mode:
    cd frontend && npm run dev

  Build frontend:
    cd frontend && npm run build
