                        HOME HUB - APPLICATION AUDIT
                    Last updated: April 26, 2026 (late evening)
================================================================================

System: Single-user, always-on home automation dashboard with ML intelligence
Stack:  FastAPI + SvelteKit + SQLite + WebSocket + LightGBM + YAMNet +
        MediaPipe + Pi-hole (Docker)
Prod:   Dell Latitude 7420 (Ubuntu 24.04) kiosk at 192.168.1.210
Dev:    Windows desktop at 192.168.1.30
Repo:   github.com/agatte/home-hub

This doc tracks the substantial work that has shipped and the substantial
work that has not. Trivial fixes (one-line bug patches, dependency pins,
small a11y nudges) are deliberately omitted -- check git log for those.

                       MAJOR WORK SHIPPED
================================================================================

The full timeline lives in git. This list is "what would I tell someone
returning to the project after a long break."

  ML FOUNDATION (Phases 1-3)
  -----------------------------------------------------------------------
  - 8 ML services in backend/services/ml/ (~2,100 LOC), all using the
    shadow-before-promote pattern with non-ML fallbacks
  - ConfidenceFusion: 5-signal weighted ensemble (process, camera,
    audio, behavioral, rule_engine) + presence voter, auto-apply at 95%
    when idle/away, override stale processes at 98% with 80% agreement
  - Accuracy-driven weight learning runs nightly at 3:30 AM via
    fusion.update_weights_from_accuracy(), driven by ml_logger
    .compute_accuracy_by_source(days=14) (main.py:502-521)
  - BehavioralPredictor (LightGBM, 10 features), AudioClassifier
    (YAMNet 521->9 classes), LightingPreferenceLearner (EMA per-light),
    MusicBandit (Thompson sampling), FeatureBuilder, MLDecisionLogger,
    ModelManager
  - Camera service: MediaPipe face + pose, zone (desk/bed) + posture
    (upright/reclined) with 15s hysteresis, opt-in via app_settings,
    frames never persisted
  - Smart screen sync: K-means dominant color (replaces naive averaging)
  - Rule engine with learned time-based mode patterns from event data
  - Event logging across 6 tables, 90-day rolling retention, fire-and-
    forget pattern with bounded retry queue
  - Live analytics dashboard: SVG confidence ring, animated signal
    cards, per-source accuracy, decision history

  LIGHTING REDESIGN (April 16 - April 25)
  -----------------------------------------------------------------------
  - Per-light spatial variation across every mode (no _uniform fallback)
  - Kitchen pair rule (L3+L4 paired in functional modes, free to
    diverge in relax / social)
  - Post-sunset ct>=333 (~3000K) warmth cutoff in evening / night
  - Colorspace exclusivity in hue_service: forces sat=0, drops hue when
    ct is set, emits sat before ct (greenish-bedroom bug killed)
  - _reconcile_effect orders state-apply before effect-stop (fixes the
    100% brightness pop on mode switch); 0.5s guard between stop/start
  - Watching tuned for projector (no D65, L2 cross-room bias, kitchen
    OFF post-evening)
  - Late-night autopilot cascade: 22:00 winddown override + LATE_NIGHT
    process-weight decay 0.6 + 23:00 mode-flip safety net
  - Zone+posture overlay: bed+reclined lowers L1/L2 across modes (not
    just watching) at evening / night / late_night; desk+watching lifts
    L2 above the projector-safe dim
  - _get_time_period() now resolves the morning ramp window to "day"
    (was falling through to "night" weekend 8-10am, weekday 6-7am)
  - Camera commits (zone, posture) cleared on sustained absence and on
    resume-from-sleeping; _apply_zone_overlay enforces a 5-min
    freshness gate so stale commits can't dim the room

  PRESENCE SYSTEM (April 19 - April 22)
  -----------------------------------------------------------------------
  - WiFi presence with two-signal detection: iPhone Shortcut webhooks
    (primary, X-Presence-Token authed) + active ARP probing (backup);
    ICMP retired (iOS WiFi power-save was producing phantom away events)
  - Arrival ceremony (light wave, weather-aware effect, music auto-play)
  - Departure fade gated by three vetoes: manual_override,
    camera-present-recently, current activity mode
  - Presence is also a fusion voter (weight 0.18, low-conf idle)

  BACKEND HARDENING (April 24 - April 25)
  -----------------------------------------------------------------------
  - Pydantic WS validation: discriminated union, range-validated,
    error frame returned for invalid input
  - Resilient lifespan shutdown: _safe_shutdown isolates per-step
    teardown failures, ordered producers -> tasks -> clients
  - Event-logger retry queue: 200-entry deque, 30s drainer, 3 attempts;
    /health exposes queue_depth + overflow
  - Heartbeat surface in /health: every poll loop publishes last_tick
    via HeartbeatRegistry; `tasks_stale` lists any whose age > 2x
    interval, `status` flips to "degraded" (HTTP stays 200). 9 loops
    instrumented; camera deregisters on disable / sleep-pause
  - Circuit breakers on Hue + Sonos: 3 consecutive failures opens
    (30s cooldown, 5s per-call timeout). /health surfaces breaker
    state alongside heartbeats — heartbeat says "loop wedged",
    breaker says "calls failing"
  - Request tracing IDs: 12-hex correlation ID per HTTP request +
    per WebSocket connection, propagated via ContextVar, stamped on
    every LogRecord by a logging Filter, echoed back via X-Request-ID
    header (or connection_status payload for WS)
  - ML predictor health surface in /health: each predictor (behavioral,
    lighting, bandit, audio, fusion, rule engine) reports
    healthy/shadow/idle/unhealthy via a HealthTrackable mixin; status
    flips to "degraded" only for non-shadow unhealthy predictors so
    silent inference failures (model load, JSON corruption, swallowed
    exceptions) become visible without false-positive paging on
    intentional shadow lanes
  - deploy.sh auto-rollback: failed post-restart health check resets
    to last-deployed SHA, re-runs install/rebuild from prior code,
    re-verifies. Exit 1 on rollback success, exit 2 if rollback
    itself failed

  BACKEND DI CLEANUP (April 26)
  -----------------------------------------------------------------------
  - Replaced 12 lifespan late-binds (`service._attr = dep` after
    construction) with constructor injection across AutomationEngine,
    RuleEngineService, MusicMapper, AmbientSoundService. main.py
    lifespan now constructs services in dependency order — weather,
    ML services, confidence fusion, music bandit, rule engine, music
    mapper, ambient sound all built before AutomationEngine and passed
    in via kwargs. PresenceService now receives a real
    confidence_fusion at construction (was previously back-filled
    because fusion didn't exist yet at presence-construction time).
    Mode-change callback registrations remain post-construction —
    those are runtime event subscriptions, not DI
  - Extracted lifespan to backend/bootstrap.py: main.py shrank from
    970 → 350 lines. Pure relocation, zero behavior change. main.py
    no longer drags in 30+ service imports — just the four needed by
    the WebSocket handler. Sets up the next refactor (splitting
    automation_engine.py)
  - Extracted per-light state calculator from automation_engine.py to
    backend/services/light_state_calculator.py: ACTIVITY_LIGHT_STATES,
    EFFECT_AUTO_MAP, LUX_CURVE, WINDDOWN_RAMP_MINUTES, BED_RECLINED_*
    constants and the pure functions that consume them
    (get_time_period, resolve_activity_state, apply_brightness_
    multiplier, apply_lux_multiplier, apply_zone_overlay, apply_
    weather_adjust, classify_weather, adjust_single_light, lerp_
    light_state, morning_ramp). Engine methods are now thin shims
    that thread engine state (schedule_config, _mode_brightness,
    camera reads, _last_lux_multiplier) through pure functions; same
    names + signatures so engine callers and 95 existing engine tests
    keep working unchanged. Hysteresis state stays on the engine,
    routed through the calculator's tuple return. Engine drops
    2,779 → 2,184 lines (-595); calculator adds 740 lines. 45 new
    unit tests in test_light_state_calculator.py exercise the pure
    functions in isolation

  SECURITY HARDENING (April 25 - April 26)
  -----------------------------------------------------------------------
  - X-API-Key on every write endpoint (POST/PUT/PATCH/DELETE) via a
    single FastAPI dependency. Localhost (kiosk colocation) and
    configured TRUSTED_LAN_IPS bypass the header check; everything
    else fails 401, missing-key deploys fail closed at 503. Presence
    webhooks keep their separate X-Presence-Token path. 69 protected
    writes across 11 routers
  - Plant App credentials no longer travel cleartext: urlparse-based
    scheme validation in PlantAppService rejects http:// at boot
    unless PLANT_APP_ALLOW_INSECURE=true is set explicitly (escape
    hatch for migration only, emits a WARNING on every login)
  - Sonos SSRF allowlist on play_uri: regex tuple gates iTunes /
    mzstatic / our own LOCAL_IP and rejects everything else.
    Validated at the route handler (400 telemetry) and inside the
    service method (defense in depth). TTS path verified compatible
  - /api/debug/query opened read-only at the engine layer (SQLite
    file:...?mode=ro), not just at the validator. Validator
    upgraded to a comment-stripping tokenizer that allows SELECT
    and WITH (CTE reads were previously rejected). Results capped
    at 1000 rows with a truncated flag

  FRONTEND PERFORMANCE + A11Y (April 24)
  -----------------------------------------------------------------------
  - Threlte + three.js (~689 KB) lazy-loaded via dynamic import when
    mode==sleeping; new MoonBackground.svelte wraps Canvas + MoonScene

  - ParallaxScene rAF loop now capped at 60fps (was uncapped on 144Hz)
  - api.js: AbortController + 8s timeout on apiGet/apiPost; HTTP
    errors surfaced via addError() instead of swallowed; parseJsonOrNull
    distinguishes empty body (HTTP 204) from malformed JSON

================================================================================
                          PRIVACY POSTURE
================================================================================

  - Camera frames: numpy arrays in memory only, overwritten each cycle.
    Never serialized, persisted, networked, or logged
  - Audio: raw PCM overwritten each 1s read. Mel-spectrograms one-way.
    Only labels persist
  - Models: aggregate statistical patterns only, no PII reconstructable
  - 90-day rolling retention on event tables
  - Full ML data wipe available from Settings (models + decisions +
    metrics)
  - All inference is local-only -- no cloud APIs
