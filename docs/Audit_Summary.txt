================================================================================
                        HOME HUB - APPLICATION AUDIT
                    Last updated: April 25, 2026
================================================================================

System: Single-user, always-on home automation dashboard with ML intelligence
Stack:  FastAPI + SvelteKit + SQLite + WebSocket + LightGBM + YAMNet +
        MediaPipe + Pi-hole (Docker)
Prod:   Dell Latitude 7420 (Ubuntu 24.04) kiosk at 192.168.1.210
Dev:    Windows desktop at 192.168.1.30
Repo:   github.com/agatte/home-hub

This doc tracks the substantial work that has shipped and the substantial
work that has not. Trivial fixes (one-line bug patches, dependency pins,
small a11y nudges) are deliberately omitted -- check git log for those.

================================================================================
                       MAJOR WORK SHIPPED
================================================================================

The full timeline lives in git. This list is "what would I tell someone
returning to the project after a long break."

  ML FOUNDATION (Phases 1-3)
  -----------------------------------------------------------------------
  - 8 ML services in backend/services/ml/ (~2,100 LOC), all using the
    shadow-before-promote pattern with non-ML fallbacks
  - ConfidenceFusion: 5-signal weighted ensemble (process, camera,
    audio, behavioral, rule_engine) + presence voter, auto-apply at 95%
    when idle/away, override stale processes at 98% with 80% agreement
  - Accuracy-driven weight learning runs nightly at 3:30 AM via
    fusion.update_weights_from_accuracy(), driven by ml_logger
    .compute_accuracy_by_source(days=14) (main.py:502-521)
  - BehavioralPredictor (LightGBM, 10 features), AudioClassifier
    (YAMNet 521->9 classes), LightingPreferenceLearner (EMA per-light),
    MusicBandit (Thompson sampling), FeatureBuilder, MLDecisionLogger,
    ModelManager
  - Camera service: MediaPipe face + pose, zone (desk/bed) + posture
    (upright/reclined) with 15s hysteresis, opt-in via app_settings,
    frames never persisted
  - Smart screen sync: K-means dominant color (replaces naive averaging)
  - Rule engine with learned time-based mode patterns from event data
  - Event logging across 6 tables, 90-day rolling retention, fire-and-
    forget pattern with bounded retry queue
  - Live analytics dashboard: SVG confidence ring, animated signal
    cards, per-source accuracy, decision history

  LIGHTING REDESIGN (April 16 - April 25)
  -----------------------------------------------------------------------
  - Per-light spatial variation across every mode (no _uniform fallback)
  - Kitchen pair rule (L3+L4 paired in functional modes, free to
    diverge in relax / social)
  - Post-sunset ct>=333 (~3000K) warmth cutoff in evening / night
  - Colorspace exclusivity in hue_service: forces sat=0, drops hue when
    ct is set, emits sat before ct (greenish-bedroom bug killed)
  - _reconcile_effect orders state-apply before effect-stop (fixes the
    100% brightness pop on mode switch); 0.5s guard between stop/start
  - Watching tuned for projector (no D65, L2 cross-room bias, kitchen
    OFF post-evening)
  - Late-night autopilot cascade: 22:00 winddown override + LATE_NIGHT
    process-weight decay 0.6 + 23:00 mode-flip safety net
  - Zone+posture overlay: bed+reclined lowers L1/L2 across modes (not
    just watching) at evening / night / late_night; desk+watching lifts
    L2 above the projector-safe dim
  - _get_time_period() now resolves the morning ramp window to "day"
    (was falling through to "night" weekend 8-10am, weekday 6-7am)
  - Camera commits (zone, posture) cleared on sustained absence and on
    resume-from-sleeping; _apply_zone_overlay enforces a 5-min
    freshness gate so stale commits can't dim the room

  PRESENCE SYSTEM (April 19 - April 22)
  -----------------------------------------------------------------------
  - WiFi presence with two-signal detection: iPhone Shortcut webhooks
    (primary, X-Presence-Token authed) + active ARP probing (backup);
    ICMP retired (iOS WiFi power-save was producing phantom away events)
  - Arrival ceremony (light wave, weather-aware effect, music auto-play)
  - Departure fade gated by three vetoes: manual_override,
    camera-present-recently, current activity mode
  - Presence is also a fusion voter (weight 0.18, low-conf idle)

  BACKEND HARDENING (April 24 - April 25)
  -----------------------------------------------------------------------
  - Pydantic WS validation: backend/schemas/ws.py, discriminated union,
    range-validated, error frame returned for invalid input
  - Resilient lifespan shutdown: _safe_shutdown helper isolates per-
    step teardown failures, ordered producers->task-cancel->client-
    close, bounded gather timeout
  - Event-logger retry queue: 200-entry bounded deque, 30s drainer,
    3 attempts per entry, threshold WARN on sustained drops; /health
    exposes event_logger_queue_depth + event_logger_overflow
  - Background-task heartbeat surface (April 25): every long-running
    poll loop publishes last_tick via HeartbeatRegistry; /health
    returns `tasks` (per-task age + interval + stale flag) and flips
    `status` to `"degraded"` when any task's age exceeds 2x its
    expected interval. HTTP stays 200 so Uptime Kuma's existing probe
    keeps working. 9 loops instrumented including the camera service
    (which deregisters on disable / sleep-pause)

  FRONTEND PERFORMANCE + A11Y (April 24)
  -----------------------------------------------------------------------
  - Threlte + three.js (~689 KB) lazy-loaded via dynamic import when
    mode==sleeping; new MoonBackground.svelte wraps Canvas + MoonScene
  - prefers-reduced-motion honored across all canvas scenes (Aurora,
    Pixel, Parallax, Moon)
  - ParallaxScene rAF loop now capped at 60fps (was uncapped on 144Hz)
  - api.js: AbortController + 8s timeout on apiGet/apiPost; HTTP
    errors surfaced via addError() instead of swallowed; parseJsonOrNull
    distinguishes empty body (HTTP 204) from malformed JSON

  TEST COVERAGE (April 24)
  -----------------------------------------------------------------------
  - Backend: 183 -> 405+ tests
    * ML services: behavioral_predictor, lighting_learner, music_bandit,
      feature_builder, ml_logger (~70 new tests)
    * ConfidenceFusion ensemble: 30 cases (staleness boundary, weight
      normalization, late-night decay, agreement-denominator stale
      exclusion, threshold edges, weight-learning math)
    * Backend hardening (Pydantic WS / lifespan / event-logger retry)
  - Frontend: 0 -> 60 tests
    * vitest + @testing-library/svelte + jsdom (55 unit / component)
    * Playwright + 5 e2e smoke (home + 3 routes + /health)
    * GitHub Actions runs vitest on push/PR (Playwright is local-only;
      see Still Open)

  INFRASTRUCTURE / OPS (carried baseline)
  -----------------------------------------------------------------------
  - Smart deploy.sh: diffs HEAD, only rebuilds what changed, health-
    checks after restart, auto-rolls back to last-deployed SHA on
    health-check failure (re-runs install/rebuild from prior code,
    re-verifies health), kiosk auto-reload via WebSocket build_id
  - Daily SQLite backup cron on Latitude (4 AM, 7-day retention)
  - GitHub Actions CI (ruff lint + pytest + vitest)
  - Pi-hole v6 in Docker (host networking) on Latitude
  - Unified PC agent supervisor (single pythonw.exe under Task
    Scheduler manages activity_detector + ambient_monitor + screen sync)
  - Custom MCP server with 25+ tools (.mcp.json)
  - Skills: /home-hub-dev, /api-audit, /ui-audit, /deploy-home,
    /project-spec
  - py-spy + httpie on Latitude for production debugging

================================================================================
                          CATEGORY DETAILS
================================================================================

The full per-category writeups have been pruned to issues that remain.
Strengths and shipped items are summarized in MAJOR WORK SHIPPED above.

  BACKEND ARCHITECTURE -- B-
  -----------------------------------------------------------------------
  - automation_engine.py at 2,750 lines absorbs every new capability
  - main.py lifespan ~350+ lines initializing 20+ services
  - Service DI by attribute assignment is invisible coupling
  - 22+ bare "except Exception" clauses on critical paths
  - No middleware for request logging / tracing
  - No circuit breaker for Hue / Sonos
  - Light IDs are strings in some places, ints in others

  FRONTEND / UI -- B+
  -----------------------------------------------------------------------
  - Modal portal duplication risk (PlantWidget + BarWidget)
  - LightCard color presets duplicated from theme.js
  - LightGrid sort runs on every reactive change (memoize)
  - Three.js DPR cap loses sharpness on 3x displays
  - Mobile touch targets: FloatingNav 40x40 at 480 breakpoint (WCAG
    wants 48x48); LightCard color swatches 20x20

  SECURITY -- C
  -----------------------------------------------------------------------
  - Zero authentication: 80+ endpoints fully open on LAN
  - Plant App integration sends credentials over HTTP
  - Preview URL injection: sonos.play_uri accepts arbitrary URLs
  - MCP query_db accepts arbitrary SELECT (UNION risk)
  - HTTP only -- blocks Cloudflare Tunnel until TLS is added

  ML / INTELLIGENCE -- A-
  -----------------------------------------------------------------------
  - BehavioralPredictor in cold start (needs 500+ events)
  - AudioClassifier still in shadow mode (needs accuracy comparison)
  - LightingPreferenceLearner overlay applied silently -- no
    ml_decisions row when the learner touches a light
  - No A/B framework for comparing ML vs rule engine decisions
  - lightgbm optional -- silent graceful degradation could hide
    predictor failures without monitoring

  PERFORMANCE -- B
  -----------------------------------------------------------------------
  - Hue polling at 1s (could be 3-5s for 6 bulbs)
  - Sonos polling at 2s (could be 5s)
  - Per-light WS broadcasts on scene activation (N sends, not batched)
  - GenerativeCanvas particle count resizes frame-by-frame during
    transitions (GC pressure)
  - MoonScene shader materials at module scope (VRAM leak risk on
    unmount/remount)

  OBSERVABILITY -- B-
  -----------------------------------------------------------------------
  - No /metrics endpoint
  - No request tracing / IDs
  - No DB health in /health
  - No ML model health alerts
  - Log timestamps missing timezone

  TESTING -- B+
  -----------------------------------------------------------------------
  - No integration tests (end-to-end mode change -> lights -> music)
  - Services without backend tests: Hue, Sonos, TTS, screen sync,
    routines, recommendation_service, library_import_service
  - Playwright e2e is local-only (needs CI runner with live backend)
  - No coverage reporting (pytest-cov not configured)

  DEVOPS / DEPLOYMENT -- B-
  -----------------------------------------------------------------------
  - No staging environment
  - No automated frontend build verification in CI (Python only)
  - Playwright e2e local-only (CI doesn't run them)

  DOCUMENTATION -- A-
  -----------------------------------------------------------------------
  - No ops runbook (troubleshooting / log inspection / recovery)
  - No changelog for breaking changes
  - No DB migration documentation (migrations inline in database.py)

================================================================================
                          PRIVACY POSTURE
================================================================================

  - Camera frames: numpy arrays in memory only, overwritten each cycle.
    Never serialized, persisted, networked, or logged
  - Audio: raw PCM overwritten each 1s read. Mel-spectrograms one-way.
    Only labels persist
  - Models: aggregate statistical patterns only, no PII reconstructable
  - 90-day rolling retention on event tables
  - Full ML data wipe available from Settings (models + decisions +
    metrics)
  - All inference is local-only -- no cloud APIs
